The Method is an open-source refinement layer for AI coding agents. Describe what you want in plain language; it turns that into a build-ready spec — domain mapped, decisions captured, threats modelled, every story testable.

It stops at the spec. Your coding tool builds from it. That's the point.

the-method — triage

How it works

From fuzzy intent to committed spec, in one conversation.

01

Describe the work

A question, a bug, a domain to map, an epic — no skill names to memorise. Triage figures out what shape you typed and only goes as deep as the work warrants.

02

Refine through interview

Specialised roles convene: scope, decisions, threat model, design, story tree. The AI brings expertise and pushes back on weak reasoning; you bring taste and the final call. Artifacts are drafted from your answers and signed off by you.

03

Build from the spec

Ready stories land in your tracker, each with acceptance criteria and a failing test that defines done. Take them to Claude Code, Cursor, or Codex to implement — the spec is now airtight.

What you get

Every output is structured, and every output is in git: scope briefs, threat models with the engineer's engagement recorded, ADRs, design docs, failing tests, a compliance manifest. Nothing important leaves your head without being recorded.

The role panel

Ten roles, each with a distinct job, quality bar, and failure mode. The Method composes them based on what the work needs — you never pick one by name. Every interviewer is a thinking partner, not a stenographer: it pushes back.

One prompt to Claude Code, Cursor, or Codex. The install runs as a guided interview that adapts the Method to your project — stack, tracker, compliance posture, existing decisions.

FAQ

Does it write my production code?

No. The Method stops at a ready spec — stories with failing tests, decisions, threat models, a domain map. Implementation happens in your own coding tool, against everything the Method produced. Separating the two is deliberate: specs are where being wrong is cheap.

Do I need a tracker?

No. Linear, Jira, and GitHub Issues are supported via MCP, but tracker.type: none keeps everything in git — the tree.yaml becomes the operational state. Without a tracker connected, refinement runs in dry-run mode and still produces all git artifacts.

Which agents does it work with?

It's built on Claude Code (agents + skills), and the artifacts it produces — markdown, YAML, failing tests — are readable by any tool. Cursor and Codex can run the install and build from the specs; session handoffs work across agents too.

Do I have to use it for everything?

No. For a five-line fix, just fix it. The Method is for work where the upstream thinking matters — decisions, designs, threat models, anything bigger than a bug.

What if it gets something wrong?

Push back — the Method assumes you will, and records every pushback. Agents are plain markdown files in your repo: if one keeps making the same mistake, edit .claude/agents/{name}.md and the change applies immediately.
